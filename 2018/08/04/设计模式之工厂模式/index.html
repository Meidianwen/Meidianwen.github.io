设计模式之工厂模式

发表于 2018-08-04 | 分类于 设计模式 | | 阅读次数

工厂模式是Java中最常用的设计模式之一。这中类型的设计模式属于创建模式，此模式提供了创建对象的最佳方法之一。通过此方法，可以不使用new关键词来创建对象，避免了把对象的实现逻辑暴露出来，使用通用的接口引用新创建的对象。

总结

刚开始并不太理解为什么需要这么做，在查了一点资料后才有点顿悟。
首先，一个对象有三个一般的职责：

  • 对象本身所具有的职责
  • 创建对象的职责
  • 使用对象的职责

所以当一个类，通过new方式来创建一个对象后，一般地，在这个类中一定还会使用创建的对象，这就造成了类的职责不是单一的，也就是说 创建对象 和 使用对象 在此类中是高耦合的，如果想使用另外一套对象，则必须修改源代码。最常用的方法是把创建对象从此类中移除，由工厂类来负责创建，这个类只负责使用，达到了工厂类不涉及对象的使用，此类不涉及对象的创建，降低了维护成本。
在所有工厂模式中，都强调一点：两个类A和B之间的关系仅仅是A创建B或者是A使用B，而不能两种关系都有，对象的创建与使用的分离也更加符合 单一职责原则，有利于对功能的复用和维护。
此外，将对象的创建和使用分离还有一个好处： 防止用来实例化一个类的数据和代码在多个类中到处都是，可以将有关创建的知识搬移到一个工厂类中
